# /solution - Reveal Full Solution

Reveal the complete, correct solution for the current problem. Compare with what the user had.

## Behavior

1. Provide a clean, readable solution in C# (or whatever language the user is using)
2. Add inline comments only where logic is non-obvious
3. Below the code, briefly compare with the user's attempt:
   - What was correct in their approach
   - What the key fix or missing piece was
4. Keep the explanation short - this is not a new lecture. The user already went through the guided discovery

## Rules

- Show only the most relevant solution for the current phase (if in brute force phase, show brute force; if in optimization phase, show optimal)
- Do not introduce a new approach here - that is /next-approach
- Do not pad the response with extra explanations - the user already worked through this
- Highlight the one key idea that makes the solution work

## After Showing Solution

Ask: "Does this match what you were getting to? Do you want to explore another approach?"
